Challenge 2: Design challenge

The useFront property

I initially navigated to the 'component folder' to locate Camera-related components. Initially, I found a similar component, which led to some confusion. However, upon further exploration, I managed to locate the camera component logic in a camera.java in components folder. The use of the useFront property immediately caught my attention. Upon reviewing the code, I identified the section responsible for camera configuration. On researching online, I discovered that the android.intent.extras.LENS_FACING_FRONT intent was deprecated starting from Lollipop MR1.

if (useFront) {

intent.putExtra("android.intent.extras.CAMERA_FACING", 1);

}

Since this intent got deprecated, it got broken. To address this issue, one potential solution would be to check the Android version and adjust the camera configuration accordingly:

if (useFront) {

if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.LOLLIPOP_MR1) {

intent.putExtra("android.intent.extras.LENS_FACING_FRONT", 1);

} else {

intent.putExtra("android.intent.extras.CAMERA_FACING", 1);

}

}




Taking photos automatically


Upon considering how to implement the solution, I realized that creating a separate component for automatic image capture would greatly benefit users. Typically, when users seek specific functionality, they look for dedicated components using kewords like auto camera. So, having a separate component would make things intuitive and keep the existing one simple. This special component will enable automatic picture-taking without users having to do anything. They can even customize how often it captures images and the total number of pictures. For App Inventor Users: Simply drag and drop the 'AutoCamera' component onto your screen. Then, adjust settings like capture frequency and total count. When designing your app, you'll find the 'AutoCamera' component alongside other non-visible ones. You can customize its behavior in the settings.


Creating the AutoCamera Component:

File Modifications: Camera.java: Extend this class to create AutoCamera.java. Add necessary properties and methods for automatic picture-taking. AutoCamera.java: Implement logic for automatic picture-taking. Utilize Android SDK features for camera control. Manage periodic image capture using a timer or background thread. Trigger the PictureTaken event after each capture.

Properties for AutoCamera: Interval (milliseconds): Set the time interval between consecutive picture captures. Use a timer or background thread to manage the interval. Max Pictures (optional): Limit the number of pictures taken automatically.

User Experience: In the designer view, users can drag and drop the AutoCamera component. They can configure properties like interval and maximum picture count.

Blocks View: Provide blocks for: StartCapturing(): Initiates automatic picture-taking. StopCapturing(): Halts the process. PictureTaken event: Triggered after each capture.
